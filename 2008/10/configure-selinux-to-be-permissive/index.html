Configure SELINUX To Be Permissive

Created2008-10-09Updated2020-08-09

Regardless of your technical opinion towards SELINUX whether it’s a necessary evil or not worth it you can either change the default state to disabled or permissive which may help trouble shoot permission errors. To modify SELINUX we need to edit the file /etc/selinux/config, do not forget to become root before modifying the files.

[root@localhost ~]# nano /etc/selinux/config

Search for the line that reads :

SELINUX=enforcing

The following alternative options are available : permissive and disabled I prefer the service to function under “permissive”.

SELINUX=permissive

Now, reboot your system and the changes should take effect.
